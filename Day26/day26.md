# Day 26 – Prior Authorization Workflow Simulator

## Project Overview

For Day 26 of the #60DayClaudeChallenge, I built a **Prior Authorization Workflow Simulator** using **HTML, CSS, and Vanilla JavaScript**. The application simulates the complete healthcare prior authorization workflow followed by providers and insurance payers in the United States.

The simulator provides an interactive, drag-and-drop experience that helps users understand how authorization requests move through different stages, how documentation affects approval decisions, and how appeals and peer-to-peer reviews work.

---

## Technologies Used

- HTML5
- CSS3
- Vanilla JavaScript
- Single File Application
- Responsive UI
- No External Libraries or Frameworks

---

## Features Implemented

- Interactive Patient → Provider → Payer workflow
- Drag-and-Drop case movement
- Multiple healthcare scenarios
  - Elective Surgery
  - MRI
  - Specialty Medication
  - Inpatient Admission
- Medical Necessity Evaluation
- Prior Authorization Document Collection
- PA Submission Process
- Payer Review
- Approval Workflow
- Pend Workflow
- Denial Workflow
- Appeal Process
- Peer-to-Peer Review
- Workflow Progress Tracker
- Days Elapsed Counter
- Efficiency Score
- Activity Log
- Educational Explanations at every stage
- Workflow Summary
- Celebration Animation after approval
- Restart / New Patient functionality
- Responsive Blue-themed User Interface

---

## Workflow Covered

1. Care Need Identified
2. Medical Necessity Evaluation
3. Document Collection
4. PA Submission
5. Payer Review
6. Decision
7. Appeal (if required)
8. Peer-to-Peer Review
9. Final Outcome
10. Workflow Summary

---

## Key Learnings

While building this simulator, I learned:

- How the Prior Authorization process works in healthcare.
- Why medical necessity documentation is important.
- How providers prepare and submit authorization requests.
- How insurance payers review clinical documents.
- Difference between Approval, Pend, and Denial.
- Why appeals and peer-to-peer reviews are important.
- How workflow visualization improves understanding of complex business processes.
- How gamification can make educational applications more engaging.
- Managing complete application state using Vanilla JavaScript without external libraries.
- Building drag-and-drop interactions using the HTML5 Drag and Drop API.


---

## Conclusion

This project provided a practical understanding of healthcare workflow automation and demonstrated how complex Prior Authorization processes can be transformed into an engaging, educational, and interactive browser-based simulator using only HTML, CSS, and Vanilla JavaScript.
