# Day 39 – PDF Splitter & Merger

## 📌 Challenge
Built a premium browser-based **PDF Splitter & Merger** as part of the **#60DaysClaudeChallenge**. The application processes PDF files entirely on the client side, ensuring privacy while providing a modern, responsive user experience.

## 🎯 Objective
Create a commercial-quality PDF utility that allows users to:
- Split PDFs using multiple methods
- Merge multiple PDF files
- Preview documents before processing
- Perform all operations locally in the browser

## 🚀 Features
- Split PDF by custom page ranges
- Split after selected pages
- Split every N pages
- Extract selected pages
- Merge multiple PDF files
- Drag-and-drop file upload
- Drag-and-drop file reordering
- PDF page preview and thumbnails
- Client-side PDF processing
- Responsive modern UI
- Dark mode interface
- Smooth animations and micro-interactions
- Keyboard shortcuts
- Accessibility support
- Works completely offline after initial library load

## 🛠️ Technologies Used
- HTML5
- CSS3
- Vanilla JavaScript
- PDF.js
- PDF-lib


## 📚 Key Learnings
- Learned how client-side PDF processing works without a backend.
- Improved understanding of PDF.js and PDF-lib integration.
- Practiced building modern drag-and-drop interfaces.
- Designed a polished productivity application with responsive UI.
- Enhanced skills in creating offline-first web applications.

## ✅ Outcome
Successfully built and tested a professional PDF Splitter & Merger capable of splitting, merging, previewing, and downloading PDFs entirely within the browser while maintaining user privacy.
